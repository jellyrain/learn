history

允许操作浏览器的曾经在标签页或者框架里访问的会话历史记录

表示当前窗口首次使用以来用户的导航历史记录。因为 history 是 window 的属性， 所以每个 window 都有自己的 history 对象

出于安全考虑，这个对象不会暴露用户访问过的 URL， 但可以通过它在不知道实际 URL 的情况下前进和后退

1. 静态属性：​

1. length 只读：​

返回一个整数，该整数表示会话历史中元素的数目，包括当前加载的页。例如，在一个新的选项卡加载的一个页面中，这个属性返回 1

2. scrollRestoration：​

允许 Web 应用程序在历史导航上显式地设置默认滚动恢复行为。此属性可以是自动的（auto）或者手动的（manual）

3. state 只读：​

返回一个表示历史堆栈顶部的状态的值。这是一种可以不必等待 popstate 事件而查看状态的方式

2. 静态方法：​

1. back()：​

在浏览器历史记录里前往上一页，用户可点击浏览器左上角的返回 (注：←) 按钮模拟此方法。等价于 history.go(-1)

2. forward()：​

在浏览器历史记录里前往下一页，用户可点击浏览器左上角的前进 (注：→) 按钮模拟此方法。等价于 history.go(1)

3. go()：​

通过当前页面的相对位置从浏览器历史记录 ( 会话记录 ) 加载页面

比如：参数为-1 的时候为上一页，参数为 1 的时候为下一页

如果当前页为第一页，前面已经没有页面了，我传参的值为-1，那么这个方法没有任何效果也不会报错

调用没有参数的 go() 方法或者参数值为 0 时，重新载入当前页面

4. pushState()：​

向当前浏览器会话的历史堆栈中添加一个状态（state）。

history.pushState(state, title[, url])

参数：

  1. state：一个对象，用于存储任意数据。这个对象会被存储在 event.state 属性中
  2. title：一个字符串，用于更新浏览器的标题
  3. url：一个字符串，用于更新浏览器的 URL

注意：浏览器不会在调用 pushState() 之后尝试加载此 URL，但可能会稍后尝试加载 URL，例如在用户重新启动浏览器之后

5. replaceState()：​

修改当前历史记录实体，如果你想更新当前的 state 对象或者当前历史实体的 URL 来响应用户的的动作的话这个方法将会非常有用

history.replaceState(stateObj, title[, url]);

参数：

  1. stateObj：一个对象，用于存储任意数据。这个对象会被存储在 event.state 属性中
  2. title：一个字符串，用于更新浏览器的标题
  3. url：一个字符串，用于更新浏览器的 URL
