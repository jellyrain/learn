原型 和 原型链

1. 原型概念：​

1. 基础概念：​

js 分为「函数对象」和「普通对象」，每个对象都有 __proto__ 属性，但是只有函数对象且「非箭头函数」才有 prototype 属性

  1. 属性 __proto__ 是一个对象【实例通过 __proto__ 隐式原型指向其原型对象】，它有两个属性，constructor 和 __proto__
  2. 原型对象有一个默认的 constructor 属性，用于记录实例是由哪个构造函数创建

prototype：显式原型

__proto__：隐式原型

2.原型：​

创建一个函数（非箭头函数），就会按照特定的规则为这个函数创建一个 prototype 属性（指向原型对象）：

  1. 所有原型对象自动获得一个名为 constructor 的属性，指回与之关联的构造函数
  2. 在自定义构造函数时，原型对象默认只会获得 constructor 属性，其他的所有方法都继承自 Object
  3. 每次调用构造函数创建一个新实例，这个实例的内部 [[Prototype]] 指针就会被赋值为构造函数的原型对象。脚本中没有访问这个[[Prototype]] 特性的标准方式，但 Firefox、Safari 和 Chrome 会在每个对象上暴露 __proto__ 属性，通过这个属性可以访问对象的原型

  1. Person.prototype 指向原型对象，而 Person.prototype.contructor 指回 Person 构造函数
  2. 原型对象包含 constructor 属性和其他后来添加的属性
  3. Person 的两个实例 person1 和 person2 都只有一个内部属性指回 Person.prototype，而且两者都与构造函数没有直接联系

3. 查找流程：​

  1. 在通过对象访问属性时，会按照这个属性的名称开始搜索
  2. 搜索开始于对象实例本身。如果在这个实例上发现了给定的名称，则返回该名称对应的值
  3. 如果没有找到这个属性，则搜索会沿着指针进入原型对象，然后在原型对象上找到属性后，再返回对应的值

2. 原型链：​

重温一下构造函数、原型和实例的关系：

每个构造函数都有一个 prototype 指向原型对象，原型对象有一个 constructor 属性指回构造函数，而实例有一个内部指针指向原型

如果原型是另一个类型的实例呢？

那就意味着这个原型本身有一个内部指针指向另一个原型，相应地另一个原型也有一个指针指向另一个构造函数。这样就在实例和原型之间构造了一条原型链。这就是原型链的基本构想

3. 模拟 new：​

  1. 创建一个空对象，作为将要返回的对象实例
  2. 将这个空对象的隐式原型，指向了构造函数的 prototype（显式原型） 属性
  3. 将这个空对象赋值给函数内部的 this 关键字
  4. 开始执行构造函数内部的代码
  5. 如果构造函数返回一个对象，那么就直接返回该对象，否则返回创建的对象
const myNew = (Fn, ...args) => {
  const obj = {}
  obj.__proto__ = Fn.prototype
  result = Fn.apply(obj, args)
  return result instanceof Object ? result : obj
}
