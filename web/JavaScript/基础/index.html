基础

1. 注释：​

功能：对程序的解释说明(方便其他人快速理解你的代码) 特点：不影响程序的编译和运行 分类：

1. 单行注释
        语法：  // 注释内容
        特点：只对本行//后的内容起到注释作用
        应用场合：对局部代码的解释

2. 多行注释
        语法： /*   多行解释内容  */
        特点： 在范围内的内容，都是注释
        应用场合：对局部代码的解释

2. 关键字：​

特点：变颜色、全小写

个数会比较多，不用限制全部背下，遇到哪个算哪个

自己起名的时候，不能使用 JavaScript 中的关键字和保留字

3. 标识符：​

在程序开发过程中，需要自己创建大量的标识符

标识符的命名规则：(必须遵守)

  1. 字母、数字、符号组成
  2. 符号只有两个 $ 和 _
  3. 数字不能作为开头
  4. 不能有空格、区分大小写
  5. 不能使用 JavaScript 中的关键字和保留字

标识符的命名规范：(建议遵守)

  1. 见名知意

    1. 类名和接口名

      每个单词的首字母都大写，形式：XxxYyyZzz 大驼峰式命名法

    2. 变量名和方法名

      从第二个单词开始首字母大写，其余字母小写，形式：xxxYyyZzz 小驼峰命名法

    3. 包名

      每一个单词都小写，单词之间使用点.分割，形式：xxx.yyy.zzz，

    4. 常量名

      每一个单词都大写，单词之间使用下划线 _ 分割，形式：XXX_YYY_ZZZ

4. 数据类型：​

数据类型说明
Null空值，表示非对象
undefined未定义的值，表示未赋值的初始化值
Number数字，数学运算的值
String字符串，表示信息流
Boolean布尔值，逻辑运算的值
Symbol（ES6）该类型的性质在于这个类型的值可以用来创建匿名的对象属性，且具有唯一性
BigInt（ES11）它可以表示任意精度格式的整数
Object对象，表示复合结构的数据集

5. 常量：​

概念：在程序执行的过程中，其值不可以发生改变的量

const final = 10

注意：如果是一个 引用类型（对象）是可以修改 引用类型 内部值

6. 变量：​

概念：在程序执行的过程中，其值可以发生改变的量 作用：用来存储数据，代表内存的一块存储区域，这块内存中的值是可以改变的 使用：

var num = 10

let num = 10

num = 10 // 不带修饰符 会挂载到 全局对象 上

注意：

  1. var 声明的变量属于函数作用域，let 和 const 声明的变量属于块级作用域
  2. var 变量可以重复声明，而在同一个块级作用域，let 变量不能重新声明，const 变量不能修改
  3. 变量提升
    1. var 声明的变量存在变量提升，即变量可以在声明之前调用，值为 undefined
    2. let 和 const 不存在变量提升，即它们所声明的变量一定要在声明后使用，否则报错
  4. 暂时性死区
    1. var 不存在暂时性死区
    2. let 和 const 存在暂时性死区，只有等到声明变量的那一行代码出现，才可以获取和使用该变量

7. 输出语句：​

console.log('hello world!')

8. 转义符：​

要求：必须写在字符串内

    \  -->  反斜杠
    \n   换行
    \"  就是一个很普通的双引号(不会被程序识别为字符串的标识)
    \t  一个制表位(tab)   占4个汉字的空间或8个字母或数字的空间,但是这个空间内可以存放内容
    \\  一个普通的斜杠

9. 类型转换：​

1. string -> number：​

Number('32') // 32

Number('Jonas') // NaN

注意：

  1. NaN 的类型是 number
  2. == 两个 NaN 为 false
  3. Number.isNaN() 能检查是否是 NaN

2. number -> string：​

String('32') // "32"

注意：

有字符串的 + 情况下 字符串后的 全部非字符串都会转换为字符串 在拼接

3. Boolean：​

1. 转换成 boolean：​

  1. false：0, '', undefined, null, NaN
  2. true：非0, 非空, {}, []

2. boolean 转出去：​

  1. string：对应的字符串，"true" "false"
  2. number：0, 1
