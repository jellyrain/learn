Node

Node 是一个接口，各种类型的 DOM API 对象会从这个接口继承。它允许我们使用相似的方式对待这些不同类型的对象

可以查找 DOM 节点，改变 DOM 节点的属性，添加和删除 DOM 节点，以及操作 DOM 节点的子节点。

1. 以下接口都从 Node 继承其方法和属性：​

Document、Element、CharacterData（它是在其他接口中被实现的，如 Text、Comment 等这些非抽象接口）、DocumentFragment

2. 节点类型：​

常量值描述
Node.ELEMENT_NODE1一个 元素 节点，例如 <p> 和 <div>
Node.TEXT_NODE3Element 或者 Attr 中实际的 文字
Node.CDATA_SECTION_NODE4一个 CDATASection，例如 <!CDATA[[ … ]]>
Node.PROCESSING_INSTRUCTION_NODE7一个用于 XML 文档的 ProcessingInstruction，例如 <?xml-stylesheet ... ?> 声明
Node.COMMENT_NODE8一个 Comment 节点。
Node.DOCUMENT_NODE9一个 Document 节点
Node.DOCUMENT_TYPE_NODE10描述文档类型的 DocumentType 节点。例如 <!DOCTYPE html> 就是用于 HTML5 的
Node.DOCUMENT_FRAGMENT_NODE11一个 DocumentFragment 节点

3. 实例属性：​

1. baseURI 只读：​

返回一个表示 base URL 的 DOMString。不同语言中的 base URL 的概念都不一样。 在 HTML 中，base URL 表示协议和域名，以及一直到最后一个'/'之前的文件目录

2. childNodes 只读：​

返回一个包含了该节点所有子节点的实时的 NodeList。NodeList 是动态变化的。如果该节点的子节点发生了变化，NodeList 对象就会自动更新

3. firstChild 只读：​

返回该节点的第一个子节点 Node，如果该节点没有子节点则返回 null

4. isConnected 只读：​

返回一个布尔值用来检测该节点是否已连接 (直接或者间接) 到一个上下文对象上，比如通常 DOM 情况下的 Document 对象，或者在 shadow DOM 情况下的 ShadowRoot 对象

5. lastChild 只读：​

返回该节点的最后一个子节点 Node，如果该节点没有子节点则返回 null

6. nextSibling 只读：​

返回与该节点同级的下一个节点 Node，如果没有返回 null

7. nodeName 只读：​

返回一个包含该节点名字的 DOMString。节点的名字的结构和节点类型不同。比如 HTMLElement 的名字跟它所关联的标签对应，就比如 HTMLAudioElement 的就是 'audio' ，Text 节点对应的是 '#text' 还有 Document 节点对应的是 '#document'

8. nodeType 只读：​

返回一个与该节点类型对应的无符号短整型的值，可能的值在节点类型查找

9. nodeValue：​

返回或设置当前节点的值

10. ownerDocument 只读：​

返回这个元素属于的 Document 对象。 如果没有 Document 对象与之关联，返回 null

11. parentNode 只读：​

返回一个当前节点 Node 的父节点。如果没有这样的节点，比如说像这个节点是树结构的顶端或者没有插入一棵树中， 这个属性返回 null

12. parentElement 只读：​

返回一个当前节点的父节点 Element。 如果当前节点没有父节点或者说父节点不是一个元素 (Element), 这个属性返回 null

13. previousSibling 只读：​

返回一个当前节点同辈的前一个节点 (Node) ，或者返回 null（如果不存在这样的一个节点的话）

14. textContent：​

返回或设置一个元素内所有子节点及其后代的文本内容

4. 实例方法：​

1. appendChild()：​

将一个节点附加到指定父节点的子节点列表的末尾处

element.appendChild(aChild)

参数：​

aChild：要追加给父节点（通常为一个元素）的节点

返回值：​

返回追加后的子节点 （aChild），除非 aChild 是一个文档片段（DocumentFragment），这种情况下将返回空文档片段（DocumentFragment）

注意：​

  1. 如果将被插入的节点已经存在于当前文档的文档树中，那么 appendChild() 只会将它从原先的位置移动到末尾（不需要事先移除要移动的节点）
  2. 如果某个节点已经拥有父节点，在被传递给此方法后，它首先会被移除，再被插入到新的位置

2. cloneNode()：​

克隆一个 Node，并且可以选择是否克隆这个节点下的所有内容。默认情况下，深度克隆

node.cloneNode([deep])

参数：​

  1. dupNode：克隆生成的副本节点
  2. deep：是否采用深度克隆，如果为 true，则该节点的所有后代节点也都会被克隆，如果为 false，则只克隆该节点本身

返回值：​

克隆生成的副本节点

注意：​

  1. 克隆一个元素节点会拷贝它所有的属性以及属性值，当然也就包括了属性上绑定的事件
  2. 不包括 JavaScript 动态绑定的事件：addEventListener() 和 node.onclick = function(event) {}

3. compareDocumentPosition()：​

比较当前节点与任意文档中的另一个节点的位置关系

node.compareDocumentPosition(otherNode)

参数：​

otherNode：用于比较位置的 Node

返回值：​

返回值是一个具有以下值的位掩码：

常量名十进制值含义
DOCUMENT_POSITION_DISCONNECTED1不在同一文档中
DOCUMENT_POSITION_PRECEDING2otherNode 在 node 之前
DOCUMENT_POSITION_FOLLOWING4otherNode 在 node 之后
DOCUMENT_POSITION_CONTAINS8otherNode 包含 node
DOCUMENT_POSITION_CONTAINED_BY16otherNode 被 node 包含
DOCUMENT_POSITION_IMPLEMENTATION_SPECIFIC32待定

4. contains()：​

来表示传入的节点是否为该节点的后代节点

node.contains(otherNode)

参数：​

  1. node: 是否包含 otherNode 节点
  2. otherNode: 是否是 node 的后代节点

返回值：​

如果 otherNode 是 node 的后代节点或是 node 节点本身。则返回 true , 否则返回 false

5. getRootNode()：​

返回上下文对象的根节点。如果 shadow root 节点存在的话，也可以在返回的节点中包含它

6. hasChildNodes()：​

返回一个 Boolean 布尔值，来表示该元素是否包含有子节点

7. insertBefore()：​

在当前节点下增加一个子节点 Node，并使该子节点位于参考节点的前面

parentNode.insertBefore(newNode, referenceNode)

参数：​

  1. parentNode：新插入节点的父节点
  2. newNode：用于插入的节点
  3. referenceNode：newNode 将要插在这个节点之前

返回值：​

函数返回被插入过的子节点；当 newNode 是 DocumentFragment 时，返回空 DocumentFragment

8. isDefaultNamespace()：​

返回一个 Boolean 类型值。接受一个命名空间 URI 作为参数，当参数所指代的命名空间是默认命名空间时返回 true，否则返回 false

9. isEqualNode()：​

返回一个 Boolean 类型值。当两个 node 节点为相同类型的节点且定义的数据点匹配时（即属性和属性值相同，节点值相同）返回 true，否则返回 false

10. normalize()：​

对该元素下的所有文本子节点进行整理，合并相邻的文本节点并清除空文本节点

在一个"规范化"后的 DOM 树中，不存在一个空的文本节点，或者两个相邻的文本节点

  1. “空的文本节点”并不包括空白字符 (空格，换行等) 构成的文本节点
  2. 两个以上相邻文本节点的产生原因包括：
    1. 通过脚本调用有关的 DOM 接口进行了文本节点的插入和分割等
    2. HTML 中超长的文本节点会被浏览器自动分割为多个相邻文本节点

11. removeChild()：​

从 DOM 中删除一个子节点。返回删除的节点

node.removeChild(child)

参数：​

  1. child：是要移除的那个子节点
  2. node：是 child 的父节点

返回值：​

删除的子节点的引用

12. replaceChild()：​

用指定的节点替换当前节点的一个子节点，并返回被替换掉的节点

parentNode.replaceChild(newChild, oldChild)

参数：​

  1. newChild：用来替换 oldChild 的新节点。如果该节点已经存在于 DOM 树中，则它首先会被从原始位置删除
  2. oldChild：被替换掉的原始节点

返回值：​

被替换掉的原始节点
