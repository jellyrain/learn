Document

Document 接口表示任何在浏览器中载入的网页，并作为网页内容的入口，也就是 DOM 树。DOM 树包含了像 <body> 、<table> 这样的元素，以及大量其他元素

它向网页文档本身提供了全局操作功能，能解决如何获取页面的 URL ，如何在文档中创建一个新的元素这样的问题

一般来说我们会使用它的实例（document）操作 DOM 树

1. 实例属性：​

1. body：​

返回当前文档的 <body> 或 <frameset> 节点

2. characterSet 只读：​

返回文档正在使用的字符集

3. compatMode 只读：​

指示文档是否以 quirks 怪异模式或 strict 严格模式呈现

4. contentType 只读：​

根据当前文档的 MIME Header，返回它的 Content-Type

5. doctype 只读：​

返回当前文档的文档类型定义（Document Type Definition, DTD）

6. documentElement 只读：​

返回当前文档的直接子节点。对于 HTML 文档，HTMLHtmlElement 对象一般代表该文档的 <html> 元素

7. documentURI 只读：​

以字符串的类型，返回当前文档的路径

8. embeds 只读：​

以列表 (list) 的类型，返回当前文档的嵌入式的元素 <embed>

9. fonts：​

返回当前文档的 FontFaceSet 接口

10. forms 只读：​

返回一个包含当前文档中所有表单元素 <form> 的列表

11. head 只读：​

返回当前文档的 <head> 元素

12. hidden 只读：​

返回一个布尔值，表明当前页面是否隐藏

13. images 只读：​

返回当前文档中所包含的图片的列表

14. implementation 只读：​

返回与当前文档相关联的 DOM 实现

15. lastStyleSheetSet 只读：​

返回最后启用样式表的名字。在设置 selectedStyleSheetSet 前值都为 null

16. links 只读：​

返回一个包含文档中所有超链接的列表

17. mozSyntheticDocument：​

返回 Boolean ，仅当此文件是合成的（例如独立图像，视频，音频文件等）时才为 true

18. plugins 只读：​

返回一个可用插件列表

19. scripts 只读：​

返回文档中所有的 <script> 元素

20. scrollingElement 只读：​

返回对文档 Element 元素的引用

21. selectedStyleSheetSet：​

返回当前正使用的样式表集

22. styleSheetSets 只读：​

返回文档上可用样式表的列表

23. timeline 只读：​

返回 DocumentTimeline 的一个实例，该实例是在页面加载时自动创建的

24. visibilityState 只读：​

返回 string 表明当前文档的可见性。可能的取值有 visible, hidden, prerender, and unloaded

2. 扩展属性：​

1. cookie：​

返回一个使用分号分隔的 cookie 列表，或设置（写入）一个 cookie

2. defaultView 只读：​

返回一个对（当前）window 对象的引用

3. designMode：​

获取或设置（让用户）编辑整个文档的能力

4. dir 只读：​

获取或设置文档的文字方向（rtl 或 ltr）

5. domain：​

获取或设置当前文档的域名

6. lastModified 只读：​

返回文档最后修改的时间

7. location 只读：​

返回当前文档的 URI

8. readyState 只读：​

返回当前文档的加载状态

9. referrer 只读：​

返回来源页面的 URI

10. title：​

获取或设置当前文档的标题

11. URL 只读：​

以字符串形式返回文档的地址栏链接

3. 实例方法：​

1. adoptNode()：​

从其他的 document 文档中获取一个节点

该节点以及它的子树上的所有节点都会从原文档删除 (如果有这个节点的话), 并且它的 ownerDocument 属性会变成当前的 document 文档，之后你可以把这个节点插入到当前文档中

document.adoptNode(externalNode)

参数：​

externalNode：将要从外部文档导入的节点

返回值：​

导入当前文档的新节点。新节点的 parentNode 是 null, 因为它还没有插入当前文档的文档树中，属于游离状态

2. createComment()：​

创建一个新的注释节点并返回

document.createComment(data)

参数：​

data：是一个字符串，包含了注释的内容

返回值：​

一个注释节点

3. createDocumentFragment()：​

创建一个新的文档片段

4. createElement()：​

用给定标签名 tagName 创建一个新的元素

document.createElement(tagName)

参数：​

tagName：指定要创建元素类型的字符串

返回值：​

新建的元素

5. createTextNode()：​

创建一个 text 节点

document.createTextNode(data)

参数：​

data：是一个字符串，包含了要放入文本节点的内容

返回值：​

新建的 text 节点

6. close()：​

结束由对文档的 document.write() 写入操作，这种写入操作一般由 document.open() 打开

7. getElementsByName()：​

根据给定的 name 返回一个在 HTML document 的节点列表集合

document.getElementsByName(name)

参数：​

name：是元素的 name 属性的值

返回值：​

一个实时更新的 NodeList 集合。当文档中有同一个 name 属性的元素被添加或移除时，这个集合会自动更新

8. getElementsByClassName()：​

返回一个包含了所有指定类名的子元素的类数组对象

当在 document 对象上调用时，会搜索整个 DOM 文档，包含根节点

document.getElementsByClassName(names)

参数：​

names：是一个字符串，表示要匹配的类名列表，类名通过空格分隔

返回值：​

一个实时集合，包含了找到的所有元素

9. getElementsByTagName()：​

返回一个包括所有给定标签名称的元素的 HTML 集合

document.getElementsByTagName(name)

参数：​

name：是一个代表元素的名称的字符串。特殊字符 "*" 代表了所有元素

返回值：​

是一个由发现的元素出现在树中的顺序构成的动态的 HTML 集合

10. getElementById()：​

返回一个匹配特定 ID 的元素。由于元素的 ID 在大部分情况下要求是独一无二的，这个方法自然而然地成为了一个高效查找特定元素的方法

document.getElementById(id)

参数：​

id：是大小写敏感的字符串，代表了所要查找的元素的唯一 ID

返回值：​

是一个 Element 对象。如果当前文档中拥有特定 ID 的元素不存在则返回 null

11. hasFocus()：​

返回一个 Boolean，表明当前文档或者当前文档内的节点是否获得了焦点

12. importNode()：​

将外部文档的一个节点拷贝一份，然后可以把这个拷贝的节点插入到当前文档中

document.importNode(externalNode, deep)

参数：​

  1. externalNode：将要从外部文档导入的节点
  2. deep：一个布尔值，表明是否要导入节点的后代节点

返回值：​

导入当前文档的新节点。新节点的 parentNode 是 null, 因为它还没有插入当前文档的文档树中，属于游离状态

13. open()：​

打开一个要写入的文档

14. querySelector()：​

返回文档中与指定选择器或选择器组匹配的第一个 Element 对象。 如果找不到匹配项，则返回 null

document.querySelector(selectors)

参数：​

selectors：包含一个或多个要匹配的选择器的 DOM 字符串，该字符串必须是有效的 CSS 选择器字符串

返回值：​

表示文档中与指定的一组 CSS 选择器匹配的第一个元素，一个 Element 对象。如果没有匹配到，则返回 null

15. querySelectorAll()：​

返回与指定的选择器组匹配的文档中的元素列表 (使用深度优先的先序遍历文档的节点)。返回的对象是 NodeList

parentNode.querySelectorAll(selectors)

参数：​

selectors：包含一个或多个要匹配的选择器的 DOM 字符串，该字符串必须是有效的 CSS 选择器字符串

返回值：​

一个静态 NodeList，包含一个与至少一个指定选择器匹配的元素的 Element 对象，或者在没有匹配的情况下为空 NodeList

16. write()：​

将一个文本字符串写入一个由 document.open() 打开的文档流

17. writeln()：​

向文档中写入一串文本，并紧跟着一个换行符
