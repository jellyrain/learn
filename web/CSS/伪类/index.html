伪类

1. :active：​

:active 伪类匹配被用户激活的元素

它让页面能在浏览器监测到激活时给出反馈。当用鼠标交互时，它代表的是用户按下按键和松开按键之间的时间

2. :checked：​

:checked 伪类表示任何处于选中状态的控件

用户通过勾选/选中元素或取消勾选/取消选中，来改变该元素的 :checked 状态

3. :disabled：​

:disabled 伪类表示任何被禁用的元素

如果一个元素不能被激活（如选择、点击或接受文本输入）或获取焦点，则该元素处于被禁用状态

4. :empty：​

:empty 伪类代表没有子元素的元素

如果一个元素不能被激活（如选择、点击或接受文本输入）或获取焦点，则该元素处于被禁用状态

5. :enabled：​

:enabled 伪类表示任何被启用的元素

如果一个元素能够被激活（如选择、点击或接受文本输入），或者能够获取焦点，则该元素是启用的

6. :first-child：​

:first-child 伪类表示在一组兄弟元素中的第一个元素

7. :first-of-type：​

:first-of-type 伪类表示在一组兄弟元素中其类型的第一个元素

8. :focus：​

:focus 伪类表示在获得焦点的元素（如表单输入）

当用户点击或触摸元素或通过键盘的 tab 键选择它时会被触发

9. :focus-within：​

:focus-within 伪类表示在一个元素获得焦点，或，该元素的后代元素获得焦点

元素自身或者它的某个后代匹配 :focus 伪类

10. :has()：​

:has() 伪类表示在一个元素，其给定的选择器参数（相对于该元素的 :scope）至少匹配一个元素

元素自身或者它的某个后代匹配 :focus 伪类

示例：

下面的选择器只会匹配直接包含 <img> 子元素的 <a> 元素：

a: has(> img);

下面的选择器只会匹配其后紧跟着 <p> 元素的 <h1> 元素：

h1: has(+ p);

11. :hover：​

:hover 伪类表示在使用指示设备悬停一个元素（没有激活它）的情况

设备指示放在元素上的时候

12. :indeterminate：​

:indeterminate 伪类表示在状态不确定的表单元素

根据上述例子（选择器）选中的元素是： <input type="checkbox"> 元素，其 indeterminate 属性被 JavaScript 设置为 true

<input type="radio"> 元素, 表单中拥有相同 name 值的所有单选按钮都未被选中时

处于不确定状态的 <progress> 元素

13. :in-range：​

:in-range 伪类表示在代表一个 <input> 元素，其当前值处于属性 min 和 max 限定的范围之内

14. :invalid：​

:invalid 伪类表示在任意内容未通过验证的 <input> 或其他 <form> 元素

15. :lang()：​

:lang() 伪类表示在基于元素语言来匹配页面元素

16. :last-child：​

:last-child 伪类表示在一组兄弟元素中的最后一个元素

17. :last-of-type：​

:last-of-type 伪类表示在一组兄弟元素中其类型的最后一个元素

18. :link：​

:link 伪类是用来选中元素当中的链接

给链接设置属性

它将会选中所有尚未访问的链接，包括那些已经给定了其他伪类选择器的链接

19. :not()：​

:not() 伪类表示在用来匹配不符合一组选择器的元素

由于它的作用是防止特定的元素被选中，它也被称为反选伪类（negation pseudo-class）

20. :nth-child：​

:nth-child 伪类从所有兄弟节点（非指定元素参与计数）中从前往后匹配处于某些位置的元素

示例：

0n+3 或简单的 3 匹配第三个元素

1n+0 或简单的 n 匹配每个元素

2n+0 或简单的 2n 匹配位置为 2、4、6、8...的元素, 可以使用关键字 even 来替换此表达式

2n+1 匹配位置为 1、3、5、7... 的元素, 可以使用关键字 odd 来替换此表达式

3n+4 匹配位置为 4、7、10、13... 的元素

21. :nth-last-child：​

:nth-last-child 伪类从所有兄弟节点（非指定元素参与计数）中从后往前匹配处于某些位置的元素

示例：

0n+3 或简单的 3 匹配第三个元素

1n+0 或简单的 n 匹配每个元素

2n+0 或简单的 2n 匹配位置为 2、4、6、8...的元素, 可以使用关键字 even 来替换此表达式

2n+1 匹配位置为 1、3、5、7... 的元素, 可以使用关键字 odd 来替换此表达式

3n+4 匹配位置为 4、7、10、13... 的元素

22. :nth-of-type：​

:nth-of-type 伪类从指定元素（非指定元素不参与计数）中从前往后匹配处于某些位置的元素

示例：

0n+3 或简单的 3 匹配第三个元素

1n+0 或简单的 n 匹配每个元素

2n+0 或简单的 2n 匹配位置为 2、4、6、8...的元素, 可以使用关键字 even 来替换此表达式

2n+1 匹配位置为 1、3、5、7... 的元素, 可以使用关键字 odd 来替换此表达式

3n+4 匹配位置为 4、7、10、13... 的元素

23. :nth-last-of-type：​

:nth-last-of-type 伪类从指定元素（非指定元素不参与计数）中从后往前匹配处于某些位置的元素

示例：

0n+3 或简单的 3 匹配第三个元素

1n+0 或简单的 n 匹配每个元素

2n+0 或简单的 2n 匹配位置为 2、4、6、8...的元素, 可以使用关键字 even 来替换此表达式

2n+1 匹配位置为 1、3、5、7... 的元素, 可以使用关键字 odd 来替换此表达式

3n+4 匹配位置为 4、7、10、13... 的元素

24. :only-child：​

:only-child 伪类匹配没有任何兄弟元素的元素

等效的选择器还可以写成 :first-child:last-child 或者 :nth-child(1):nth-last-child(1) 当然,前者的权重会低一点

25. :only-of-type：​

:only-of-type 伪类匹配没有其他相同类型的兄弟的元素

26. :optional：​

:optional 伪类匹配表示任意没有 required 属性的 <input>，<select> , <textarea> 元素

27. :out-of-range：​

:out-of-range 伪类表示在代表一个 <input> 元素，其当前值处于属性 min 和 max 限定的范围之外

28. :read-only：​

:read-only 伪类表示在元素不可被用户编辑的状态（如锁定的文本输入框）

29. :read-write：​

:read-write 伪类表示在一个元素（例如可输入文本的 input 元素）可以被用户编辑

30. :required：​

:required 伪类表示在表示任意设置了 required 属性的<input>，<select> , <textarea>元素

这个伪类对于高亮显示在提交表单之前必须具有有效数据的字段非常有用

31. :root：​

:root 伪类匹配文档树的根元素

对于 HTML 来说，:root 表示 <html> 元素，除了优先级更高之外，与 html 选择器相同

32. :target：​

:target 伪类 代表一个唯一的页面元素 (目标元素)，其 id 与当前 URL 片段匹配(url 指向当前元素 id)

一般来说是配合 a 标签使用

<div id="container"></div>

<!-- 点击 a 标签 .container:target 就能生效 -->
<a href="#container"></a>

33. :valid：​

:valid 伪类表示内容验证正确的<input> 或其他 <form> 元素

对于 HTML 来说，:root 表示 <html> 元素，除了优先级更高之外，与 html 选择器相同

34. :visited：​

:visited 伪类表示用户已访问过的链接

35. :placeholder-shown：​

:placeholder-shown 在 <input> 或 <textarea> 元素显示 placeholder text 时生效
