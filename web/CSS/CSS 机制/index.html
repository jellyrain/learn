CSS 机制

1.CSS 优先级：​

CSS 优先级以此递减（选择器相同，看选择器数量：越多越高，不能越级）：

  1. !important 最高优先级
  2. 内联样式
  3. id 选择器
  4. class 选择器
  5. 伪类选择器
  6. 属性选择器
  7. 伪元素选择器
  8. 标签选择器
  9. 通用选择器

计算规则：​

条件：没有 !important 和 内联样式

(a, b, c)

  • a：ID 选择器的个数
  • b：类选择器、属性选择器 以及 伪类选择器 个数之和
  • c：标签选择器 和 伪元素选择器的个数 之和

  1. 按 a、b、c 的顺序依次比较大小，大的则优先级高，相等则比较下一个

  2. 若最后两个的选择符中 a、b、c 都相等，则按照 覆盖原则 来判断，使用最后一个属性

2. CSS 继承：​

1. 继承属性：​

当元素的一个继承属性 （inherited property）没有指定值时，则取父元素的同属性的计算值 computed value (en-US)。只有文档根元素取该属性的概述中给定的初始值（initial value）（这里的意思应该是在该属性本身的定义中的默认值）

2. 非继承属性：​

当元素的一个非继承属性(在 Mozilla code 里有时称之为 reset property )没有指定值时，则取属性的初始值 initial value（该值在该属性的概述里被指定）

3. 通用属性值：​

值描述
inherit关键字使得元素获取其父元素的计算值
initialCSS 关键字将属性的初始（或默认）值应用于元素
unset如果这个属性本来有从父级继承的值（这个属性默认可以继承，且父级有定义），则将该属性重新设置为继承的值，如果没有继承父级样式，则将该属性重新设置为初始值。换句话说，在第一种情况下（继承属性）它的行为类似于 inherit ，在第二种情况下（非继承属性）类似于 initial
