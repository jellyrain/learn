分桶表

分区提供一个隔离数据和优化查询的便利方式。不过，并非所有的数据集都可形成合理的分区。对于一张表或者分区，Hive 可以进一步组织成桶，也就是更为细粒度的数据范围划分

分桶是将数据集分解成更容易管理的若干部分的另一个技术

分区针对的是数据的存储路径；分桶针对的是数据文件

1. 先创建分桶表：​

1. 创建表：​

create table stu_buck(id int, name string)
clustered by(id) -- 表内必须有分桶字段
into 4 buckets -- 拆分为4个桶（文件）
row format delimited fields terminated by '\t';

2. 查看表结构：​

desc formatted stu_buck;

3. 导入数据到分桶表中：​

load 方式

load data inpath  '/student.txt' into table stu_buck;

insert 方式

insert into table stu_buck select * from student_insert;

4. 查询分桶的数据：​

select * from stu_buck;

5. 分桶规则：​

Hive的分桶采用对分桶字段的值进行哈希，然后除以桶的个数求余的方 式决定该条记录存放在哪个桶当中

2. 分桶表操作需要注意的事项：​

  1. reduce的个数设置为-1,让Job自行决定需要用多少个reduce或者将reduce的个数设置为大于等于分桶表的桶数

  2. 从hdfs中load数据到分桶表中，避免本地文件找不到问题

  3. 不要使用本地模式

3. 抽样查询：​

对于非常大的数据集，有时用户需要使用的是一个具有代表性的查询结果而不是全部结果。Hive可以通过对表进行抽样来满足这个需求

语法: 

TABLESAMPLE(BUCKET x OUT OF y)

查询表stu_buck中的数据

select * from stu_buck tablesample(bucket 1 out of 4 on id);

注意：

  1. x值为第几个桶开始
  2. y 是table（表）总bucket（桶）数的 倍数或者因子（table 总共分了4份，当y=2时，抽取(4/2=)2个 bucket 的数据，当y=8时，抽取(4/8=)1/2个bucket 的数据）
  3. x的值必须小于等于y的值
